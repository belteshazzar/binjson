B+ Tree (WebAssembly) with OPFS

This page demonstrates the WebAssembly B+ tree (BPlusTree from binjson-wasm.js). The tree is a durable, append-only, sorted key→value store persisted to a single OPFS file. Because it needs a FileSystemSyncAccessHandle, all tree operations run inside a Web Worker (public/worker.js); this page just sends it messages.

How it's used

Every button below drives this exact API inside the worker (see the matching bplustree-* cases in public/worker.js): 
import { BPlusTree } from '@belteshazzar/binjson/wasm';

// syncHandle is a FileSystemSyncAccessHandle (Web Worker only)
const tree = new BPlusTree(syncHandle, /* order */ 4);
await tree.open();

tree.add('alpha', { id: 1 });     // insert or update
tree.add(42, [1, 2, 3]);          // number or string keys
const v = tree.search('alpha');   // -> { id: 1 }  (undefined if absent)
const range = tree.rangeSearch(1, 100);   // sorted entries in [min, max]
tree.delete(42);
const all = tree.toArray();       // [{ key, value }, ...] sorted by key
tree.size(); tree.getHeight();    // stats

await tree.compact(destSyncHandle); // rewrite, dropping stale history
await tree.close();

Tree file

Keys are read as numbers when they look numeric, otherwise as strings. Values are parsed as JSON when possible (e.g. {"id":1}, 42, true), falling back to a plain string.
